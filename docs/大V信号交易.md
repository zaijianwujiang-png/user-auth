# 原始需求：大 V 信号自动交易系统

## 这是个什么东西

一个自动化交易系统：抓取特朗普在 Truth Social 上的最新发帖，用 AI 解析成交易信号并做多层交叉验证，通过风控检查后在币安（Binance）自动下单交易加密货币。

## 主要用户是谁

我自己（单用户系统，个人量化工具，不对外提供服务）。

## 想要的核心功能

1. **数据采集**：持续监听特朗普的 Truth Social 账号（@realDonaldTrump），发现新帖尽快抓到（目标延迟 1 分钟以内）；断线、被限流要能自动恢复并告警
2. **信号提取**：用 LLM（Claude API）把帖子解析成结构化信号：涉及的资产（BTC/ETH/DOGE 等）、方向（利多/利空/无关）、置信度；要能识别"与加密货币无关"的帖子并直接跳过
3. **交叉验证**（任一层否决就不交易）：
   - 去重与旧闻检测：转发旧消息、重复内容不触发
   - 双模型独立判断：两个不同模型结论一致才通过
   - 行情验证：拉币安实时价格/成交量，若市场已明显反应（price-in）则放弃
   - 事实核查：联网搜索确认消息真伪（防盗号、假消息）
4. **风控**：单笔仓位上限（占账户余额百分比）、日亏损熔断、同一币种冷却期、自动止损止盈；所有参数可配置
5. **下单执行**：通过币安 API 现货市价单下单（用 ccxt）；支持三种模式切换：
   - 通知模式：只推送建议，不下单
   - 半自动：推送后人工确认才下单
   - 全自动：验证通过直接下单
6. **通知与监控**：每个环节（新帖、信号、验证结果、成交、熔断、系统异常）都推送到 Telegram；每笔决策的完整链路落库可回查
7. **复盘**：记录每笔交易的信号来源和盈亏，能统计胜率和累计收益

## 有什么限制 / 已知条件

- 交易所：币安，仅现货，不碰合约杠杆（至少 v1/v2 不碰）
- 先在币安测试网（testnet）模拟跑，胜率数据可看之后才切真实账户
- API key 安全：真实账户 key 只开现货交易权限，禁止提币，配 IP 白名单；key 不进代码库
- Truth Social 没有公开 API，采集方案要评估（轮询网页/RSS 镜像/第三方聚合服务），优先合规稳定的方案
- 全链路有秒级~分钟级延迟，策略定位是中低频波段，不抢瞬时行情
- 初期默认「通知模式」，全自动是最后阶段才开
- 部署：7×24 运行（服务器/云函数均可，沿用现有 AWS 也行）

## 建议的版本切分（供拆版本参考）

- v1：采集 + AI 信号提取 + 交叉验证 + Telegram 通知（完全不碰钱）
- v2：接币安测试网自动下单 + 风控 + 交易记录
- v3：真实账户（半自动起步）+ 复盘统计 + 完整监控告警
