# change-password — 需求规格

## 概述

允许已登录用户用旧密码验证后修改为新密码。

## 项目信息

- 项目名: user-auth
- 架构类型: 单体 Serverless（Python Lambda + API Gateway + DynamoDB）

## 需求版本

| 日期       | 版本 | 说明     |
| ---------- | ---- | -------- |
| 2026-06-17 | v1   | 初始需求 |

## 用户故事

- 作为已登录用户，我想用旧密码验证后设置新密码，以便定期更换密码保障账号安全。

## 功能需求

1. [F-001] 已登录用户携带有效 token 调用 `POST /change-password`，提交 `oldPassword` 和 `newPassword`。
2. [F-002] 系统校验 token，无效/缺失返回 401。
3. [F-003] 系统校验旧密码与库中 Password Hash 匹配，不匹配返回 401。
4. [F-004] 系统校验新密码满足强度要求（≥8 位且含字母和数字），不满足返回 422。
5. [F-005] 校验通过后，用 bcrypt 重新哈希新密码并更新到数据库。
6. [F-006] 新密码不得与旧密码相同（相同返回 422）。
7. [F-007] 前端提供"修改密码"表单（需先登录）。

## 非功能需求

- 安全: 旧/新密码均不得出现在响应或日志中；新密码用 bcrypt 加盐哈希存储。
- 兼容性: 复用现有 auth/ 五模块，不破坏注册/登录/me 现有行为。

## 验收标准

- [ ] [AC-001] 有效 token + 正确旧密码 + 合规新密码 → 200，密码被更新，旧密码失效、新密码可登录。
- [ ] [AC-002] 无 token 或 token 失效 → 401。
- [ ] [AC-003] 旧密码错误 → 401。
- [ ] [AC-004] 新密码不满足强度 → 422。
- [ ] [AC-005] 新密码与旧密码相同 → 422。

## 依赖

- 现有 feature user-auth（注册/登录/me、auth/ 五模块、Users 表）。
